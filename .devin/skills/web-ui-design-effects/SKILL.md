# Premium UI Design & Motion Effects

> **Goal:** Build interfaces that feel designed by a studio, not generated by a template. Use real motion, intentional typography, custom color, and scroll/cursor micro-interactions. Everything here targets **React + Next.js + Tailwind CSS v4 + Motion/Framer Motion**.

---

<critical_requirements>

## CRITICAL: Before Using This Skill

1. **Read `CLAUDE.md` first** for project naming, `import type`, and `cn()` conventions.
2. **Do not paste generic Tailwind classes** (`bg-gradient-to-r`, `rounded-lg shadow-md`, `transition-all`) directly into marketing sections. Wrap them in section tokens or components.
3. **Respect `prefers-reduced-motion`**. Every CSS or Framer Motion animation must be skippable.
4. **Keep effects opt-in**. Add motion to *one* hero element, *one* CTA, and *one* card grid per page. Overuse looks cheap.
5. **All third-party components must be copy-and-own** (clone the repo, copy the component into `components/effects/`, adapt to our tokens). Do **not** install opaque `npm` UI kits without review.

</critical_requirements>

---

## When to use

- Landing pages, marketing sections, pricing, testimonials, or dashboards that look “AI-generic” (blue gradient hero, 3-column feature cards, Inter-700 headings, `transition-all`).
- Adding premium polish: smooth scroll, cursor micro-interactions, text reveals, bento grids, glassmorphism, aurora/mesh backgrounds.
- Re-skinning `shadcn/ui` components with custom color tokens and motion.

## When NOT to use

- Core admin/data-table screens where density and speed matter more than flourish.
- User-facing exam/test interfaces that must be distraction-free (PTE test player).
- Where accessibility (keyboard nav, screen readers, reduced motion) is compromised.

## Anti-AI generic design rules

Based on the “AI sameness” pattern (blue-indigo gradient, 3-col features, carousels, `rounded-lg shadow-md`):

| Anti-pattern | Fix |
|---|---|
| Primary hue 200–290 (blue/indigo/violet) | Move primary hue to 28–45 (warm) or 140–180 (mint/teal); keep accent high-saturation but off-axis. |
| Pure white background | Use a warm/cool tinted base (`bg-stone-50`, `bg-slate-950`, `bg-[#fafaf9]`). |
| Inter-700 headings everywhere | Pair a display serif with a clean sans (e.g. `font-serif` for H1, `font-sans` for body). |
| 3 equal columns of icon+heading+text | Use asymmetric bento grids (`5fr 3fr` or `2fr 1fr`), overlapping cards, or split-feature layouts. |
| `transition-all duration-300` | Use specific, staggered motion (e.g. `0.6s cubic-bezier(0.16,1,0.3,1)`). |
| Generic hero image/mockup | Use a real screenshot, a shader/gradient background, or a product UI video loop. |
| Abstract feature labels | Replace with concrete outcomes: “Chấm 10.000+ câu PTE tự động” thay vì “AI Powered”. |

### OKLCH color token strategy

- Define `--primary`, `--primary-foreground`, `--accent`, and `--muted` in `globals.css`.
- Avoid raw `blue-500`. Use OKLCH custom properties so colors feel intentional in dark/light modes.
- Example warm PTE palette: primary `oklch(0.65 0.18 35)`, accent `oklch(0.7 0.2 145)`, muted `oklch(0.55 0.05 250)`.

## Integration patterns

### 1. Section component skeleton

Every new section should be a standalone component in `app/sections/` or `components/sections/`. Keep layout, animation, and content separate.

```tsx
// components/sections/hero-section.tsx
'use client'
import { motion } from 'framer-motion'
import { cn } from '@/lib/utils'

export function HeroSection({ title, subtitle, cta }: HeroSectionProps) {
  return (
    <section className="relative overflow-hidden bg-[#fafaf9] py-32">
      <AuroraBackground />
      <div className="container mx-auto px-4">
        <motion.h1
          initial={{ opacity: 0, y: 40 }}
          whileInView={{ opacity: 1, y: 0 }}
          viewport={{ once: true }}
          transition={{ duration: 0.7, ease: [0.16, 1, 0.3, 1] }}
          className="font-serif text-6xl font-medium tracking-tight text-stone-900"
        >
          {title}
        </motion.h1>
      </div>
    </section>
  )
}
```

### 2. Motion token file

Create `lib/motion.ts` to keep easing/duration consistent:

```ts
export const easeOutExpo = [0.16, 1, 0.3, 1]
export const easeInOutQuart = [0.76, 0, 0.24, 1]

export const fadeUp = {
  initial: { opacity: 0, y: 24 },
  animate: { opacity: 1, y: 0 },
  transition: { duration: 0.6, ease: easeOutExpo },
}
```

### 3. Smooth scroll setup (Lenis + GSAP)

```tsx
// app/providers.tsx or layout.tsx
'use client'
import { ReactLenis } from 'lenis/react'
import { useEffect } from 'react'
import gsap from 'gsap'
import { ScrollTrigger } from 'gsap/ScrollTrigger'

gsap.registerPlugin(ScrollTrigger)

export function Providers({ children }: { children: React.ReactNode }) {
  return <ReactLenis root>{children}</ReactLenis>
}
```

### 4. Spotlight card pattern

```tsx
// components/effects/spotlight-card.tsx
'use client'
import { useRef, useState } from 'react'
import { cn } from '@/lib/utils'

export function SpotlightCard({ className, children }: SpotlightCardProps) {
  const ref = useRef<HTMLDivElement>(null)
  const [pos, setPos] = useState({ x: 0, y: 0, opacity: 0 })

  const handleMove = (e: React.MouseEvent) => {
    if (!ref.current) return
    const rect = ref.current.getBoundingClientRect()
    setPos({ x: e.clientX - rect.left, y: e.clientY - rect.top, opacity: 1 })
  }

  return (
    <div
      ref={ref}
      onMouseMove={handleMove}
      onMouseLeave={() => setPos((p) => ({ ...p, opacity: 0 }))}
      className={cn('relative overflow-hidden rounded-2xl border bg-white p-6', className)}
    >
      <div
        className="pointer-events-none absolute -inset-px transition-opacity duration-500"
        style={{
          opacity: pos.opacity,
          background: `radial-gradient(400px circle at ${pos.x}px ${pos.y}px, rgba(var(--primary-rgb), 0.12), transparent 40%)`,
        }}
      />
      <div className="relative z-10">{children}</div>
    </div>
  )
}
```

## Recommended workflow

1. **Pick one effect per page** from the tables above.
2. **Copy the source** into `components/effects/` and re-skin with project tokens.
3. **Update `globals.css` tokens** (add `--primary-rgb` if using spotlight, `--aurora-*` if using aurora).
4. **Wrap the app in `ReactLenis`** only for marketing pages; keep exam/test pages native-scroll.
5. **Run `pnpm build`** and check Lighthouse for layout shift (CLS); motion should not cause CLS.
